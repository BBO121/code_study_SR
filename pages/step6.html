div와 span에 대해 알아보자!

<div> : divide (분할) 를 줄여서 만든 대형공간 Tag. display : block

  • web2.0 시절에 태어나 대부분의 공간분할 문제를 해결해주던 Tag
  • 현재도 굉장히 많이 사용되기 때문에 무분별한 사용에 주의해야한다
  • 어느공간에도 제약이 없는 tag = html5의 구조와 의미를 규칙적으로 표현하기 힘들다

<span> : span(잘 늘어나다) 라는 의미의 소형 공간 tag. display : inline

  • 글자 일부를 묶을 때 사용
  • 줄바꿈이 되지 않음
  • 의미는 없고, 일부 스타일을 적용하거나 css,js를 적용할때 사용


semantic 태그에 대해 알아보자!

<section> : 하나의 주제를 묶는 큰 영역 태그

section 에는 제목(h)을 넣어줘야 한다. (권장)

  • 복합적인 content group (관련된 내용을 하나의 주제로 묶는 공간)
  • 일반적인 시스템에서는 body의 직계에서, 한 페이지 단위를 구성하는 개념이기 때문에, h2가 거의 고정이라고 생각하면됨
  • 이례적으로 onepage의 경우 각 항목별로 section사용

<article> : article = 기사 / 아티클 자체로 이해가 될 수 있고, 독립적으로 취급 될 수 있는 문서에 사용되는 tag

article 에는 제목(h)을 넣어줘야 한다. (권장)

  • 신문기사, 블로그 포스트, 인스타 피드 등에 사용 가능
  • time tag, cite tag 등을 같이 사용하게 된다
  • section 안에서 사용될 경우 > h3
  • 독립적인 콘텐츠로 사용될 경우 > h2

<time> : 날짜와 시간을 나타내는 태그

2026년 6월 29일 10시 42분 

        <time datetime="2026-06-29T10:42:00+09:00">2026년 6월 29일 10시 42분</time>
  • time 안에 기입하는 날짜와 시간을 사람에게 전달하는 목적
  • 이것을 기준으로 표현방식을 재현하여 태그안에 출력한다 (ex > 2026년 6월 29일...)
  • 자동으로 시간을 출력하는건 아니고, 내가 원하는 시간을 표시하는것 뿐!

<cite> 저작권 관련 표기 명칭

  • 제목, 인용한 대상, 작성자, 그림명 등등

<aside> 전체적으로 사용하는 global 중 좌, 우 측에에 있는 menu, bannerBar, quickMenu, FloatingAction...등의 content가 들어가는 aside


<footer>

저작권정보, 회사주소, 증명/신고내용, 법률관련정보, 결제가 이루어지는 곳이면 반품/환불/교환 정책등이 들어가는 하단 공통 영역(global)

  • <p> 저작권 정보
  • <address> 회사주소(일상생활에 사용하는 주소, 이메일 주소, 전화번호 등등 컨택할 수 있는 모든 수단)
  • <p> or <blockquote> 증명/신고내용
  • <a href="#">개인정보취급방침</a>
  • <a href="#">;이용약관</a>
  • <a href="#">반품/환불/교환정책</a>
    • #은 href 비어있으면 오류나서 넣어둔것임!!


HTML5 이후에 추가된 태그를 알아보자!

<figure> : figure = 흉상, 동상 / 상징적인 의미를 담은 image의 경우 사용할 수 있는 tag

확정구조 img를 쓰고 figcaption 쓰고 끝나야함!


        <figure>
            <img src="https://placehold.co/150" alt="undefined image">
            <figcaption>아직 정의되지 않은 이미지</figcaption>
        </figure>
  • image의 비중이 text보다 많이 느껴질 경우 사용
  • 이미지가 더 중요한 느낌으로 텍스트는 이미지를 보조하는 느낌
  • img tag의 alt 대용으로 figcaption이 사용 가능 (웹 접근성 측면에서도 좋음)

<progress> : 나타내고픈 진행상황을 실시간으로 나타낼 때 사용하는 tag


        <progress value="20" max="100"></progress>
  • 보통 max 100은 바꿀일은 없음
  • value = 사용값 / max = 최대값

<meter> : 현재 사용량이나 소모된 량 등의 정지된 산출값에 사용하는 tag

80% 

        <meter value="80" min="10" max="100" low="20" high="90"></meter>80%
  • 로우와 하이가 있다는 것 = 위험을 감지할 수 있다는 개념
    너무 낮아서 위험! 너무 높아서 위험! 이런게 가능. 전압이라던가...
  • value = 현재값, min = 최소값, max = 최대값, low = 낮다의 기준범위, high = 높다의 기준범위, optimum= 적당하다의 기준범위

<details> 접기/펼치기가 가능한 tag

이렇게 크게 메뉴를 쓰고 (누르기가넝)

여기 안에 세부 내용을 쓰는거에용!


        <details>
            <summary>이렇게 크게 메뉴를 쓰고 (누르기가넝)</summary>
            <p>여기 안에 세부 내용을 쓰는거에용!</p>
        </details>
  • summary 는 details 에만 들어갈 수 있다
  • summary 에는 무엇을 사용해도 상관없다 display : block
