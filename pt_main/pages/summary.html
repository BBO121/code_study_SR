p태그에 대해 알아보자!

<b> [bold 진하게]
접근성 문제가 해결이 안되어서 잘 안씀
스크린리더에서 맨앞의 글자에 강조를 함		 <strong> [strong 진하게]
청각적 효과가 있음
<i> [i 기울기]
외국어, 전문기술용어, 손글씨 등
분류적인 성격을 띄는 단어나 문장, 앞뒤 문맥과 구분되는 텍스트		 <em> [em 기울기]
청각적 효과가 있음
스크린리더에서 성조를 바꿈
<mark> [mark 색상강조]
시각적 강조만 담당하고 청각적 강조는 없음
그래서 중요도가 너무 높은곳에 쓰면 안됨 (예: 글씨색 배경색 등) > 접근성때문에
<sup> [위]첨자
superScript, 일반적인 사용이 없고 대부분 화학 수학 물리학 식등에서 사용
즉 빈도수가 낮다		 <sub> [아래]첨자
subScript, sup와 동일
<s> [s 취소선]
실제 취소나 삭제를 의미하지는 않음		 <del> [delete 취소선](실제 data상 삭제)
실제 취소, 삭제를 의미함
쇼핑몰에서 할인되기 전 금액 같은거 표현할 때 사용
<u> 밑줄
사람은 강조라고 느낄수있지만 html상에서는 단순히 밑줄일뿐 중요하다는 의미는 없다
주관적 기준에서는 강조지만, 객관적 기준에서는 강조가 아닌것
<big> [<big>일부를 크게<big>표현해]
사용할 수 없는 태그
사용 빈도수가 적거나 심각한 오류를 야기하는 태그는 사라질 수 있다		 <small> [일부를 작게 표현해]
같은 공간 내에서 일부를 작게 표현해야 할 때
  • <hr> : horizontal Rule / 단순한 수평선이 아니라 선을 기준으로 위, 아래 주제 자체가 바뀐다는 뜻
  • <blockquote> : block 상태 인용부
  • 이것이 block 상태의 인용부 입니다!
    block 은 태어나자마자 너비를 100으로 가집니다!
  • <q> : inline 의 인용문
  • 이것이 q 인용문입니다! " " 이 따옴표를 따로 쓰지 않아도 알아서 써진답니다!!

block 과 inline 이란?

display : block; (위에서부터 떨어지는 태그들) > inline-block; > inline; (글자처럼 한줄로 쭉 있는)

[ block ]

  • 태어나자마자 가로 너비 100%를 가진다
  • 항상 새로운 줄에서 시작한다
  • 블록 태그 안에는 블록, 인라인 태그 보두 들어갈 수 있다
  • 대표태그 : div, h1~h6, p, ul, ol, li, tabel, blockquote

[ inline ]

  • 내용(content)만큼만 너비를 가진다
  • 줄바꿈이 일어나지 않는다
  • 문장 안에서 사용된다
  • 대표태그 : a, strong, em, b, i, q, sub

기억하자!!

기능태그 vs 마크업 태그
[ 기능 태그 (Void Element) ]
혼자 기능을 수행하는 태그
닫는 태그가 없다!
ex> br, hr, img, input...
[ 마크업태그 (Markup)]
내용을 감싸서 구조와 의미를 만드는 태그
시작 태그와 종료태그가 있고 내용을 감싼다!
ex> h1, p, div
