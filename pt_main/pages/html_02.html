list 태그에 대해 알아보자!

<ul> : unordered list 의 약자
  • 속성값(bullet type)은 3개 circle, disc, square가 있으나 모두css에서 처리해야함
  • 레이아웃을 구성 할 때 같은 형태가 3회 이상 반복되고, 순서가 중요하지 않은 요소를 만들때 사용한다

[ ui>li 의 예시 ]

  • unorderedlist1
  • unorderedlist2
  • unorderedlist3

                    <ul>
                        <li type="circle">unorderedlist1</li>
                        <li type="disc">unorderedlist2</li>
                        <li type="square">unorderedlist3</li>
                    </ul>

<ol> : ordered list 의 약자

  • 속성값(bullet type)은 5개+@ A, a, l, i, 1(decimal) 등이 있으나 모두css에서 처리해야함
  • 레이아웃을 구성 할 때 같은 형태가 3회 이상 반복되고, 순서가 중요한 목록를 만들때 사용한다

[ ol>li 의 예시 ]

  1. orderedList1
  2. orderedList2
  3. orderedList3
  4. orderedList4
  5. orderedList5

                    <ol>
                        <li type="A">orderedlist1</li>
                        <li type="a">orderedlist2</li>
                        <li type="I">orderedlist3</li>
                        <li type="i">orderedlist4</li>
                        <li type="decimal">orderedlist5</li>
                    </ol>

<dl> : definition list 의 약자

  • dt (Definition Term) : 제목
  • dd (Definition Description) : 설명
  • dt 하나에 dd를 여러 개 사용할 수 있다
  • dt와 dd는 여러개 작성할 수 있다

[ dl>dt+dd 의 예시 ]

HTML
Hyper Text Markup Language
웹페이지를 만드는 언어
CSS
Cascading Style Sheets
웹페이지를 꾸며주는 언어
JavaScript
웹페이지에 동작을 추가하는 언어

                    <dl>
                        <dt>HTML</dt>
                        <dd>Hyper Text Markup Language</dd>
                        <dd>웹페이지를 만드는 언어</dd>
                        <dt>CSS</dt>
                        <dd>Cascading Style Sheets</dd>
                        <dd>웹페이지를 꾸며주는 언어</dd>
                        <dt>JavaScript</dt>
                        <dd>웹페이지에 동작을 추가하는 언어</dd>
                    </dl>

기억하자!

확정구조 : ul, ol은 확정구조이기 때문에 직계 자손인 li 만 들어가야한다!

이미지 태그에 대해 알아보자!

경로 = Path = 파일이 있는 위치를 적는 방식

절대경로 (A.Path = Absolute Path)

현재 존재하는 소프트웨어적 최상위 기준으로부터 시작하여 모든 경로를 기술하는 방식 > 싹다 적으라는 뜻

  • ex1. 컴퓨터 : C:/Users/kingh/OneDrive/Desktop/SR/project/code_study_SR/pages/step2.html
  • ex2. 웹 개발 환경 : /(root)/pages/step2.html
    1. (root) : 그 폴더의 가장 최상위 폴더
  • ex3. 고객 방문 환경 : DNS (ex. http://www.thespesieslab.com)
    1. DNS - Domain Naming Service

상대경로 (R.Path = Ralaive Path)

현재파일을 기준으로 파일의 위치를 작성하는 방법
현재 파일에서 목적지까지 이동하는 경로만 적는다

  • ex1. 현재 이미지 파일 불러오기 > 같은 선상에 있는 "images"폴더를 적고 파일명만 적으면 끝
    "images/file_example_JPG_2500KB.jpg"
  • 단, 상위폴더를 칭해야 할때 = "../" 한번 사용 할 때마다 한단계 상위

<img> 태그 : 이미지를 불러오는 태그 


                        <img src="../images/image_sample.jpg" alt="늘어져버린쬬이" style="width: 30%;">
  • src : "나는 가만히 있을테니까 너가 와~!!"가져와!
  • href : "내가 갈테니까 기다려~연결해!
  • alt : aleternaver Text = 대체 텍스트
             시각장애인들은 이미지를 볼 수 없기 떄문에 이미지의 내용을 전달하기 위함.
             안정화된 최대 자수는 255자 이내 권장, 이상 초과 될 경우 longdesc라는 attr이 있었으나 현재는 사용불가로 전환됨.
             figure / figcaption tag를 사용해야함.

attr 이란 ?

attr(Attribute) = 태그의 속성
태그의 추가 정보를 주거나 동작을 설정하는 역할

attr 예시 > src, href, alt, id, class, title, width, height, style 등
