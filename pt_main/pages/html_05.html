input type="radio"

입력 태그 타입 radio는 라디오 버튼(or컨디션 (이거나 조건))

  • name을 같은 이름으로 여러 항목을 지정시 그룹화 된다.
  • label을 이용하여 해당 항목과 버튼을 연결해줄 필요가 있다.
  • value 값설정으로 전달할 값을 넣어 둘 수 있으나, 비워두어도 상관없다.
  • checked 를 이용하여 페이지가 로드 될때 초기 선택값을 넣어 둘 수 있다.
    cheackbox와는 달리 하나의 항목만 선택 가능하다. 

                            <label for="btn_radio1">radio1</label>
                            <input type="radio" name="radioGroup" id="btn_radio2">
                            <label for="btn_radio2">radio2</label>
                            <input type="radio" name="radioGroup" id="btn_radio3">
                            <label for="btn_radio3">radio3</label>
                            <input type="radio" name="radioGroup" id="btn_radio4">
                            <label for="btn_radio4">radio4</label>

input type="checkbox"

  • 입력태그타입 checkbox는 다중 선택을 지원 하는 input이며, label을 이용하여 해당 항목과 연결해야 한다.
  • value 값은 기존 input 과 동일
  • checked 옵션 역시 radio input 과 동일

                                <input type="checkbox" id="btn_chk1">
                                <label for="btn_chk1">check1</label>
                                <input type="checkbox" id="btn_chk2">
                                <label for="btn_chk2">check2</label>
                                <input type="checkbox" id="btn_chk3">
                                <label for="btn_chk3">check3</label>
                                <input type="checkbox" id="btn_chk4">
                                <label for="btn_chk4">check4</label>

<select>

  • select는 선택항목의 목록형 단축 메뉴 형식으로 만드는 입력 양식
  • 항목 수 만큼의 option이 필요하고, 이 안의 value로 값을 전달한다.
  • optgroup으로 중간 그룹을 설정 할 수 있다.
  • radio, checkbox처럼 외형 변경이 어렵다.

                                <label for="selectjob">직업 선택 :</label>
                                <select id="selectjob">
                                    <optgroup label="구직자">
                                        <option>무직</option>
                                        <option>학생</option>
                                    </optgroup>
                                    <optgroup label="근로자">
                                        <option>전문직</option>
                                        <option>공무원</option>
                                    </optgroup>
                                </select>

input type="reset"

reset은 해당 폼의 모든 양식 내용을 초기화(아예 없는 상태) 시킬 때 사용한다.
반드시 form 안에 위치해야함!!!!!

  • 주의사항은 pre와 마찬가지로 입력한 그대로를 보여주기 때문에, 초기 데이터 삽입시 공백부분에 유의해야한다.

textare 주의점!

html5 이후에 나온 input 태그를 알아보자!

HTML5 Input's

input type="email"

type 을 email로 설정하면, 모바일에서 해당 필드에 focus 되었을 때, 키보드를 열면서 영문과 @버튼이 생긴다.
placeholder 로 예시로 적어둘 수 있다.(ie10이하 지원불가)

  • required : 반드시 입력해야 하는 필수입력
  • readonly : 읽기전용 (server에 저장이 된적이 있는 data를 불러오기) - 저장된적 있던 정보 - ex 아이디
  • disabled : 비활성화 (입력자체를 막아 server에 저장 자체가 불가능함.)
  • enabled : disabled의 counter - 활성화(입력가능 = 전송시 server저장가능)

                                <label for="userEmail">사용자 이메일 :</label>
                                <input type="email" id="userEmail" required readonly>

input type="url"

type 을 url로 설정하면, 모바일에서 해당 필드에 focus 되었을 떄, 키보드를 열면서 영문과 .com 버튼이 생기고
누르고 있으면 타 도메인으로 변경 가능 


                                <label for="UserURL">웹사이트 :</label>
                                <input type="url" id="UserURL" required>

input type="tel"

type을 tel로 설정하면 한줄로 전화번호 입력받는 양식 (-필수제외)
placeholder는 시각적정보만 담기 때문에 만약 label없이 사용할 경우 같은 내용의 title 필수. 


                                <label for="userPN">전화번호:</label>
                                <input type="tel" id="userPN" placeholder="-를 뺀 숫자만 입력하세요.">

input type="number"

상,하 버튼(스피너)으로 숫자를 기입하거나 입력받는 박스.
min : 최저 / max : 최대 / step : 상승값 / value : 초기값
절대 숫자입력 대용으로 사용하면 안됨. 반드시 수량 관련 부분에만 사용 할 것 


                                <label for="qty">수량 :</label>
                                <input type="number" id="qty" min="1" max="10" step="1" value="1">

input type="range"

슬라이드바로 범위 중 하나의 값을 선택할 수 있는 인풋 


                                <label for="sizeSet">사이즈 :</label>
                                <input type="range" id="sizeSet" min="250" max="280" step="10" value="250">

input type="date" : 연,월,일 선택
input type="datetime-local" : 연,월,일,시,분 선택
input type="time" : 시간 선택
input type="month" : 월 선택
input type="week" : 주 선택

input:date,datetime-local,time,month,week 는 i10 이하 지원 불가 


                                <label for="typeDate">연,월,일 선택 :</label>
                                <input type="date" id="typeDate">

                                <label for="typeDateTime">연,월,일,시,분 선택 :</label>
                                <input type="datetime-local" id="typeDateTime">

                                <label for="typeTime">시간 선택 :</label>
                                <input type="time" id="typeTime">

                                <label for="typeMonth">월 선택 :</label>
                                <input type="month" id="typeMonth">

                                <label for="typeWeek">주 선택 :</label>
                                <input type="week" id="typeWeek">

input type="search"

search는 검색어만 전문으로 입력받는 필드.
X(자체리셋) 버튼이 존재하고 자동완성기능(autocomplete)지원 


                                <label for="searchField">검색 :</label>
                                <input type="search" id="searchField">

input type="color"

olor chip을 기준으로 대상을 찾게 해주는 인풋 


                                <label for="typeColor">색상선택 :</label>
                                <input type="color" id="typeColor">

input type="submit"

폼(form)의 내용을 서버로 전송하는 버튼
required가 있는 입력란이 비어 있으면 제출되지 않는다. 


                            <input type="submit" value="전송">

input type="reset"

폼의 입력 내용을 초기 상태로 되돌리는 버튼
입력했던 글자나 선택했던 값을 모두 초기화 


                            <input type="reset" value="취소">
보통 이 두 input은 fieldset 밖에 쓰기도 한다!
